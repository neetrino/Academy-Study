# Контекст-пакеты для типовых задач

**Курс:** AI-Coding в Cursor  
**День 3:** Что давать AI, что не надо

Контекст — это валюта. Лишний контекст ухудшает ответы. Ниже наборы файлов для типовых задач.

---

## Шаблон контекст-пакета

Для каждой задачи опиши:

| Поле | Описание |
|------|----------|
| Задача | Тип задачи (рефакторинг, API, багфикс и т.д.) |
| Включать | Какие файлы давать AI (@file, @folder) |
| Не включать | Что не давать (чтобы не перегружать) |
| Правила | Нужны ли `.cursor/rules` (обычно да) |

---

## Примеры (заполни под свой проект)

### 1. Рефакторинг компонента

| Поле | Значение |
|------|----------|
| **Задача** | Рефакторинг UI-компонента |
| **Включать** | Сам компонент, его стили, родитель (где используется) |
| **Не включать** | Другие компоненты, утилиты, конфиги |
| **Правила** | Да (стиль, стек) |

**Пример:** `@src/components/Button/Button.tsx` `@src/components/Button/Button.css` `@.cursor/rules/`

---

### 2. Добавление API endpoint

| Поле | Значение |
|------|----------|
| **Задача** | Добавить новый endpoint |
| **Включать** | Роуты, контроллер, модель (если есть), существующий endpoint как пример |
| **Не включать** | UI, другие модули |
| **Правила** | Да |

---

### 3. Багфикс UI

| Поле | Значение |
|------|----------|
| **Задача** | Исправить баг в интерфейсе |
| **Включать** | Компонент с багом, родитель, связанные хуки/стейт |
| **Не включать** | Весь репозиторий |
| **Правила** | Да |

# Чек-лист: Chat vs Agent/Composer

**Курс:** AI-Coding в Cursor  
**День 2:** Выбор режима под задачу

---

## Когда Chat

| Вопрос | Если да → Chat |
|--------|----------------|
| Нужно только объяснить? | Да |
| Нужен ответ/код в тексте, без изменений файлов? | Да |
| Нужно сгенерировать фрагмент кода для копирования? | Да |
| Задача: «объясни», «какая разница», «как работает»? | Да |

**Примеры задач для Chat:**
- Объясни этот код
- Какая разница между X и Y?
- Напиши функцию сортировки (для копирования)
- Какой паттерн здесь используется?

---

## Когда Agent/Composer

| Вопрос | Если да → Agent/Composer |
|--------|-------------------------|
| Нужно изменить 2+ файлов? | Да |
| Нужен рефакторинг или добавление фичи? | Да |
| AI должен править файлы, а не писать в чат? | Да |
| Нужны команды в терминале (тесты, линтер)? | Да |

**Примеры задач для Agent/Composer:**
- Добавь компонент X и подключи в Y
- Рефакторинг модуля auth
- Добавь API endpoint и тесты
- Исправь баг в 3 файлах

---

## Вопросы для самопроверки

Перед тем как писать промпт, ответь:

1. Нужно ли AI **менять файлы**? Да → Agent. Нет → Chat.
2. Сколько файлов затронет задача? 2+ → Agent.
3. Нужен ли diff для ревью? Да → Agent.
4. Достаточно ли получить код/ответ в чате? Да → Chat.

---

## Быстрое правило

> **Только объяснить** → Chat.  
> **Нужно менять код** → Agent/Composer.

# Архитектурный контракт

**Курс:** AI-Coding в Cursor  
**День 6:** Архитектурные рамки для AI

Документ для AI и команды: структура проекта, слои, запреты.

---

## Структура папок

```
[Заполни под свой проект]

src/
├── components/     # UI-компоненты (только отображение)
├── pages/          # Страницы (сборка компонентов)
├── services/       # Бизнес-логика, API-вызовы
├── utils/          # Утилиты, чистые функции
├── hooks/          # React hooks (или аналог)
├── types/          # Типы, интерфейсы
└── ...
```

---

## Слои и зависимости

| Слой | Что содержит | Кто может импортировать |
|------|--------------|-------------------------|
| UI | components, pages | — |
| Services | Бизнес-логика, API | UI |
| Utils | Чистые функции | Services, UI |
| Types | Типы | Любой |

**Правило:** Импорты только вниз. UI не импортирует напрямую из utils (если есть services).

---

## Запреты (AI и человек)

- [ ] **Бизнес-логика в UI** — вся логика в services/utils
- [ ] **Импорты вверх** — нижний слой не импортирует верхний
- [ ] **Глобальное состояние** — только через явные контексты/сторе
- [ ] **Прямые вызовы API в компонентах** — только через сервисы

---

## Стиль

- Компоненты: один компонент — один файл
- Сервисы: один сервис — один модуль
- Именование: по `.cursor/rules`

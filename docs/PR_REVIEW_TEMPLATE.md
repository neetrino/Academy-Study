# Шаблон Code Review (с AI)

**Курс:** AI-Coding в Cursor  
**День 10:** AI как второй глаз, не как судья

AI предлагает замечания — человек фильтрует. Не всё замечание AI важно.

---

## Чек-лист для ревьюера

### Перед ревью

- [ ] Прочитать описание PR: что сделано, как проверить, риски
- [ ] Запустить локально (если возможно)
- [ ] Линтер и тесты проходят

### Во время ревью (с AI)

- [ ] Запросить AI-ревью: «Проверь этот PR на [безопасность/стиль/логику]»
- [ ] Пройти по замечаниям AI: **реальное** vs **шум**
- [ ] Для каждого замечания: принять / отклонить с обоснованием

### Что считать реальным

| Тип | Пример |
|-----|--------|
| Безопасность | Секрет в коде, невалидированный ввод |
| Баг | Ошибка в логике, edge case не обработан |
| Архитектура | Нарушение слоёв, логика в UI |
| Стиль/соглашения | Несоответствие правилам проекта |

### Что считать шумом

| Тип | Пример |
|-----|--------|
| Субъективное | «Можно переименовать» без причины |
| Устаревшее | AI не знает контекст проекта |
| Избыточное | «Добавь комментарий» там, где код ясен |

---

## Как отвечать на замечания AI

**Принять:**
> Согласен, [причина]. Исправляю.

**Отклонить:**
> Не согласен: [причина]. В нашем проекте [контекст]. Оставляю как есть.

---

## Итог ревью

- [ ] Критичные замечания исправлены
- [ ] Некритичные: приняты или отклонены с обоснованием
- [ ] PR готов к мержу (или возвращён на доработку)

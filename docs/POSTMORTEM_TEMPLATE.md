# Шаблон Postmortem (разбор инцидента)

**Курс:** AI-Coding в Cursor  
**День 15:** Инциденты — triage → hotfix → postmortem

---

## Структура Postmortem

### 1. Краткое описание

- **Дата/время:** 
- **Длительность:** 
- **Серьёзность:** [Critical / High / Medium / Low]

### 2. Что произошло

[Описание инцидента — симптомы, что увидели пользователи/мониторинг]

### 3. Triage (на момент инцидента)

- **Затронутые:** [пользователи, сервисы, функции]
- **Приоритет:** [почему именно такой]
- **Решение:** [hotfix / rollback / workaround]

### 4. Корневая причина

[Почему произошло — без обвинений, факты]

### 5. Хронология

| Время | Событие |
|-------|---------|
| | Алерт / обнаружение |
| | Triage |
| | Hotfix / action |
| | Восстановление |
| | Postmortem |

### 6. Hotfix (что сделали)

[Минимальное изменение для восстановления. Не «заодно рефакторинг»]

### 7. Что сделать, чтобы не повторилось

- [ ] [Action item 1]
- [ ] [Action item 2]
- [ ] [Action item 3]

### 8. Уроки

[Что узнали, что улучшить в процессе]

---

## Правила

- **Blameless** — фокус на процессе, не на людях
- **Факты** — что произошло, без домыслов
- **Action items** — конкретные, с владельцем и дедлайном

# Работа с AI в команде

> **Курс:** AI-Coding в Cursor (VibeCoding → Production)  
> **Назначение:** Правила и ограничения для работы с AI-ассистентом в учебном проекте.

---

## 1. Когда использовать Chat, когда — Agent/Composer

| Режим | Когда использовать | Примеры |
|-------|--------------------|---------|
| **Chat** | Объяснения, разбор кода, ответы на вопросы, генерация фрагментов | «Объясни этот код», «Какая разница между X и Y?», «Напиши функцию сортировки» |
| **Agent / Composer** | Изменения в 2+ файлах, рефакторинг, добавление фич | «Добавь компонент X и подключи в Y», «Рефакторинг модуля auth» |

**Правило:** Если нужно **только объяснить** — Chat. Если нужно **менять файлы** — Agent/Composer.

---

## 2. Что AI запрещено делать без явного согласия человека

- [ ] **Коммитить** в git
- [ ] **Пушить** в удалённый репозиторий
- [ ] **Менять** критические конфиги (prod, env, secrets)
- [ ] **Деплоить** на production
- [ ] **Удалять** тесты, линтер, проверки типов
- [ ] **Менять** зависимости без обсуждения (package.json, requirements.txt и т.д.)
- [ ] **Отключать** pre-commit хуки или CI
- [ ] **Вносить** изменения в архитектурные решения без ADR

## 3. Конфликты и разногласия: куда смотреть

| Ситуация | Источник ответа |
|----------|-----------------|
| Споры по архитектуре | `docs/ARCHITECTURE.md`, ADR в `docs/adr/` |
| Правила проекта | `.cursor/rules/`, `docs/AI_WORKFLOW.md` |
| Стиль и стек | `.cursor/rules/`, линтер-конфиг |
| Что можно/нельзя AI | Этот документ, раздел 2 |

---

## 4. Контроль реальности

**AI иногда ошибается. Особенно уверенно.**

Перед принятием изменений всегда проверять:

1. **Запуск** — проект собирается и работает
2. **Линтер** — без ошибок
3. **Тесты** — проходят (если есть)

Не доверять «вроде работает». Проверить самому.

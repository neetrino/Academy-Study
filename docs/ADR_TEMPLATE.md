# Шаблон ADR (Architecture Decision Record)

**Курс:** AI-Coding в Cursor  
**День 7:** Документация и ADR

ADR — краткий документ о принятом архитектурном решении.

---

## Структура

```markdown
# ADR-001: [Краткое название]

**Статус:** Принято / Отменено / Заменено  
**Дата:** YYYY-MM-DD

## Контекст
[Почему нужно было принять решение. 1–2 абзаца.]

## Решение
[Что решили. 1–2 абзаца.]

## Последствия
[Плюсы, минусы, что учесть при разработке.]
```

---

## Пример

### ADR-001: Хранение env-переменных

**Статус:** Принято  
**Дата:** 2024-02-01

#### Контекст
Нужно хранить API keys, URL сервера и т.д. Нельзя держать в коде.

#### Решение
- Используем `.env` для локальной разработки
- `.env` в `.gitignore`
- В проде — переменные окружения (Vercel/Heroku/env vars)
- Документируем обязательные переменные в `README`

#### Последствия
- Плюс: секреты не в репо
- Минус: надо настраивать env на каждом окружении
- Обязательно: добавить `.env.example` с placeholder'ами

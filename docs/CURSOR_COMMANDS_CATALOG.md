# Каталог команд Cursor

**Курс:** AI-Coding в Cursor  
**День 13:** Team/Project Commands

Команды стандартизируют действия. Ниже примеры и как создать.

---

## Как создать команду

1. Cursor Settings → Rules / Commands
2. Создать новую команду
3. Указать имя и промпт
4. Сохранить

---

## Рекомендуемые команды

### 1. Generate PR description

**Название:** `Generate PR description`

**Когда использовать:** После изменений, перед созданием PR

**Промпт (пример):**
```
По diff изменений в проекте сгенерируй описание PR в формате:

## Что сделано
[кратко]

## Как проверить
[шаги]

## Риски
[если есть]

## Связанные задачи
[если есть]
```

---

### 2. Write tests for file

**Название:** `Write tests for file`

**Когда использовать:** При добавлении нового файла/функции

**Промпт (пример):**
```
Напиши unit-тесты для текущего файла.
- Проверь нормальные случаи и edge cases
- Используй [Jest/Vitest]
- Тесты должны ловить регресс
- Не мокать лишнее
```

---

### 3. Security scan checklist

**Название:** `Security scan checklist`

**Когда использовать:** Перед PR с чувствительными изменениями

**Промпт (пример):**
```
Проверь код по security-чек-листу:
- Ввод и валидация
- Секреты (нет в коде)
- Auth/authorization
- Зависимости (npm audit)
- Rate limit для API

Выдай отчёт: что проверено, есть ли проблемы.
```

# Security-чек-лист для PR

**Курс:** AI-Coding в Cursor  
**День 8:** Security-минимум

Проверять перед каждым PR. Не выпускать уязвимости «потому что стажёр».

---

## 1. Ввод и валидация

- [ ] Все пользовательские входы **валидируются**
- [ ] Нет SQL/NoSQL injection (параметризованные запросы)
- [ ] Нет XSS (санитизация вывода, CSP)
- [ ] Загрузка файлов: проверка типа, размера, расширения

---

## 2. Секреты

- [ ] Никаких секретов в коде
- [ ] Никаких секретов в репозитории
- [ ] `.env` в `.gitignore`
- [ ] API keys, токены — только через env

---

## 3. Доступы (auth / authorization)

- [ ] Защищённые роуты проверяют auth
- [ ] Роли/права проверяются на бэкенде (не только на UI)
- [ ] Нет «скрытых» эндпоинтов без проверки

---

## 4. Зависимости

- [ ] `npm audit` / `pnpm audit` без критичных уязвимостей
- [ ] Известные уязвимости в зависимостях исправлены или задокументированы

---

## 5. API

- [ ] Rate limit на публичные/критичные эндпоинты
- [ ] CORS настроен корректно
- [ ] Чувствительные данные не в URL (GET с секретами)

---

## 6. Общее

- [ ] Ошибки не раскрывают внутреннюю структуру (стек-трейсы только в логах)
- [ ] HTTPS в проде

---

*Найди 5 проблем в коде — зафиксируй и исправь.*

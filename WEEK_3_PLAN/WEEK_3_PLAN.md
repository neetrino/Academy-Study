# День 11 — Планирование: epic → задачи → промпты

## Цель

Из одной фичи сделать декомпозицию на 6–10 атомарных задач с промптами.

---

## Теория (изучить перед практикой)

| Тема | Суть |
|------|------|
| Epic → задачи | Большую фичу разбить на атомарные задачи |
| DoD (Definition of Done) | У каждой задачи — чёткие критерии готовности |
| Промпты к задачам | Каждой задаче — промпт для AI (по шаблону из Недели 1) |

---

## Практика — чеклист

- [ ] Выбрать одну фичу (epic)
- [ ] Декомпозировать на **6–10 задач** (атомарных)
- [ ] Для каждой задачи: описание, DoD, промпт для AI
- [ ] Проверить: задачи можно делать параллельно или по очереди, зависимости понятны

---

## Критерии проверки (день засчитан, если)

- [ ] Задачи **атомарные**, с DoD
- [ ] Есть таск-шаблоны в `docs/TASK_TEMPLATES.md`

# День 12 — Agent + Terminal: безопасная автоматизация

## Цель

Понимать, что выполняет Agent в терминале, и уметь откатить. Определить рутину перед PR.

---

## Теория (изучить перед практикой)

| Тема | Суть |
|------|------|
| Agent + Terminal | Agent может запускать команды, тесты, линтер, сборку |
| Безопасность | Всегда проверять, что выполняет Agent, не принимать слепо |
| Откат | Уметь отменить изменения (git reset, stash) |

**Ссылка:** [Terminal \| Cursor Agent](https://cursor.com/docs/agent/terminal)

- [ ] Убедиться: умею откатить (git)
- [ ] Составить **«скрипт рутины»** — что прогоняем перед PR

## Цель

---

## Теория (изучить перед практикой)

| Тема | Суть |
|------|------|

# День 15 — Инциденты: когда «в проде горит»

## Цель

Triage → hotfix → postmortem. Приоритеты правильные, фикс минимальный.

---

## Теория (изучить перед практикой)

| Этап | Суть |
|------|------|
| **Triage** | Оценить серьёзность, кто затронут, приоритет |
| **Hotfix** | Минимальный фикс, не «пока чиним — заодно переделаем» |
| **Postmortem** | Что произошло, почему, что сделать, чтобы не повторилось |

---

## Практика — чеклист

- [ ] Симуляция инцидента: лог ошибки + алерт + частичный контекст
- [ ] Triage: оценить приоритет, затронутых пользователей
- [ ] Hotfix: минимальное изменение для восстановления
- [ ] Postmortem: написать `POSTMORTEM.md` по шаблону

---

## Критерии проверки (день засчитан, если)

- [ ] Приоритеты **правильные**
- [ ] Фикс **минимальный** (не «заодно рефакторинг»)
- [ ] Есть `docs/POSTMORTEM_TEMPLATE.md`

---
